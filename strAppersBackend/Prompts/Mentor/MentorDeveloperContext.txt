LOCAL DEVELOPMENT ‚Äî RUNNING THE BACKEND ON YOUR MACHINE
‚Ä¢ The provisioned backend is configured for Railway deployment: it reads a `PORT` environment variable and binds to `0.0.0.0:PORT`. When `PORT` is not set, it defaults to port **8080** ‚Äî which may already be in use locally. **Always tell the student to set the PORT when running locally.**
‚Ä¢ Language-specific local run commands (use the one that matches the project tech stack):
  - **C# (.NET):** `$env:PORT=5000; dotnet run` (PowerShell) or `PORT=5000 dotnet run` (bash/Mac)
  - **Node.js:** `PORT=3000 npm run dev` or `PORT=3000 node index.js`
  - **Python:** `PORT=8000 uvicorn main:app --reload` or `PORT=8000 python app.py`
  - **Ruby:** `PORT=3000 ruby app.rb` or `PORT=3000 rails server`
  - **Java:** `SERVER_PORT=8000 ./mvnw spring-boot:run` or set `server.port=8000` in `application.properties`
  - **PHP:** `PORT=8000 php -S localhost:8000 -t public` (or equivalent for the framework)
  - **Go:** `PORT=8000 go run .`
‚Ä¢ **After the build, the terminal goes quiet ‚Äî that is normal.** The server is running. The app suppresses most startup output. Tell the student to open their browser at `http://localhost:<PORT>` (e.g. `http://localhost:5000`) to confirm it's up. Do NOT say the app crashed just because the terminal went silent.
‚Ä¢ If the student gets "address already in use" or "port already in use": a previous run is still active in the background. Tell them to stop it first ‚Äî on Windows: `Stop-Process -Name <process-name> -Force` (e.g. `Stop-Process -Name Backend -Force` for C#, `Stop-Process -Name node -Force` for Node). Then re-run.
‚Ä¢ Remind the student to open the URL in the **browser address bar** ‚Äî not by typing it into the terminal (it will fail with "command not found" if run in PowerShell or bash).
