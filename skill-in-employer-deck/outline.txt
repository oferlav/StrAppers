SKILL-IN — EMPLOYER PRESENTATION OUTLINE
=========================================
11 slides · Employer Overview · 2026


SLIDE 1 — TITLE
  Headline:   "The Skills Marketplace."
  Sub:        AI profiles for employers and job seekers alike.
              When the fit is real — both sides find out.
  Pills:      AI-powered · Skills-based · Both sides of the market


SLIDE 2 — THE PROBLEM  (01 — The Problem)
  Headline:   "The market is broken on both sides."
  Left col (Companies):
    - Job descriptions are wish lists — vague, generic, copied from last year
    - Resumes measure what people claim, not what they do
    - Interviews test nerves and preparation, not real work quality
    - Every bad hire costs 6–18 months of salary to discover and fix
  Right col (Job Seekers):
    - Certifications prove study time, not ability to execute
    - Portfolio projects are polished demos — staged, not observed
    - No signal for communication, delivery pace, or collaboration style
    - No way to say "I work like your team works"
  Quote:      "Both sides are guessing. The market needs a common language —
               one that AI can read on both sides."


SLIDE 3 — THE SKILLSET  (02 — The New Unit)
  Headline:   "Not a resume. Not a job description. A Skillset."
  Left (Company Skillset — blue):
    - Work patterns — pace, consistency, collaboration style
    - Tech stack & tools — what they actually use day to day
    - Communication style — async vs sync, depth, meeting cadence
    - Culture signals — ownership, initiative, quality
    - Competency benchmarks — what "good" means here, not in general
    Tagline:  "Senior Backend Engineer at Acme Corp" — not a generic job spec.
  Right (Candidate Skillset — green):
    - Delivery record — how they perform under sprint pressure
    - Code quality — real commits, real reviews, real merges
    - Communication behaviour — team engagement, stakeholder handling
    - Problem-solving approach — blockers, ambiguity, initiative
    - Role fit signals — PM, dev, designer — patterns across sprints
    Tagline:  "Works like a mid-senior backend dev at a fast-moving product company."


SLIDE 4 — COMPANY SIDE  (03 — Company Side)

    2. Analyze   — Pick 3–5 top performers. AI reads work patterns.
    3. Calibrate — Review AI scores. Add ratings. Set dimension weights.
    4. Skillset is live — Matching begins. Courses shaped.
  Right (Example card — "Senior Backend Engineer · Acme Corp"):

    - Code Quality & Reviews:  79 — Clean PRs, avg 3.2 review comments
    - Communication Patterns:  71 — Async-first, thread-heavy, low meeting dependency
    - Culture & Stack Tags:    Ownership-driven · Remote-native · Fast iteration
                                Node.js · PostgreSQL · Docker · AWS

SLIDE 6 — PLUGIN ARCHITECTURE  (05 — Plugin Architecture)
  Headline:   "Inside the plugin."
  Sub:        How orchestration, connectors, and privacy validation work together.
  Main diagram (Employer Machine — local):
    Config layer:   CLAUDE.md · /generate-skillset · connector-config.json (credentials never leave)
    Orchestrator:   Claude AI — reads context, calls tools, asks for human ratings, applies scoring
    MCP servers:
      Connector MCP (yellow):  fetch_tasks → Jira API
                                fetch_communication → Slack API
                                fetch_artifacts → GitHub API
                                🔒 Privacy filter: strips content, keeps metrics
      Assessment MCP (teal):   get_student_board · run_gap_analysis · submit_skillset
                                ✓ Calls Skill-in API only
    External:
      Anthropic API (purple) — receives: normalized metadata only
                               (counts, ratios, frequencies · no message content · no code)
      Skill-in Backend (green) — receives: anonymized scores only
                                  (0–100 per dimension · no names · no PII)
  Privacy legend (right panel):
    Stays on machine:  API credentials · Slack message content · Jira descriptions ·
                       GitHub code diffs · Employee names & emails
    To Anthropic:      Task counts & completion rates · message word counts ·
                       commit frequency & PR ratios · timestamps & metadata
    To Skill-in:       Aggregated scores (0–100) · human rating anchors ·
                       dimension weights · company slug & role name


SLIDE 7 — CANDIDATE SIDE  (06 — Candidate Side)
  Headline:   "Not a certificate. A verified work profile."
  Sub:        Candidates earn their Skillset by doing — not by studying.
  Left (3-step flow):
    1. Shallow filter course      — Short role-based, establishes baseline fit
    2. Employer-shaped course     — Deeper course built from real Skillsets
    ✓  Candidate Skillset earned  — Months of real work, AI-scored from actual behaviour
  Right (What gets assessed):
    - Sprint delivery — tasks closed, on time, to spec
    - Code quality — real commits, PRs, code reviews
    - Communication — team, stakeholders, AI customer
    - Design thinking — spec quality, user story depth
    - Collaboration — async vs sync, initiative, blocker handling
    Callout: "A Skillset is a longitudinal work record — patterns built across weeks
              of real project pressure."


SLIDE 8 — THE LOOP  (07 — The Loop)
  Headline:   "Both sides make each other more valuable."
  Left (cycle):
    🏢 Employer submits Skillset   → defines excellence for this role
    📚 Platform shapes a course    → project, stack, criteria trained toward this standard
    👤 Candidates take the course  → real projects, verified Skillset earned
    ⚡  Matching engine fires       → Skillsets matched, both sides alerted
  Right (Network effects):
    1. More Skillsets  → richer, more varied courses
    2. More courses    → more candidates with verified profiles
    3. More candidates → faster, more accurate matches
    4. More matches    → more employers join

    📈 Network effects compound   — More Skillsets → better courses → more candidates → more matches
