## Obsolete vs client-fix (recorded policy)

- **Current frontend:** no required changes; **no new backend** endpoints to “fix” audit gaps unless you explicitly ask.
- **Mark a base44 function / call pattern obsolete** only when **100% sure** the **specific backend URL** it uses has **no** matching route in `strAppersBackend` (grep controllers). Otherwise label **verify** (e.g. auth may live outside this API).

| Call / function | 100% no matching backend? | Classification |
|-----------------|---------------------------|----------------|
| **`GET /api/Mentor/use/branch-status`** (used by `checkBranchStatus`; also first leg of `getBoardGitStatuses`) | **Yes** — not on `MentorController` | **Obsolete pattern** for any **direct** client: use `is-open-branch` + `pr-status` instead. *(`getBoardGitStatuses` as a whole is not obsolete — only the branch-status leg.)* |
| **`GET /api/StudentProjects`** (`getAssignedStudents`) | **Yes** — no `api/StudentProjects` controller | **Obsolete URL**; a future client must use a real allocation/list API (e.g. `ProjectAllocation` / `Students`) after you pick one. |
| **`GET .../api/Boards/use/use/count/...`** (`getBoardCountByProjectId`) | **No** — backend has **`GET /api/Boards/use/count/{projectId}`**; base44 has a **typo** (double `use`) | **Client-fix**, not “missing backend”; not obsolete. |
| **`/api/apps/auth/callback`** (`googleAuth`) | **Not verified** on StrAppersBackend | **Do not** mark obsolete here; confirm where auth actually runs. |

$pattern = '/api/[^`"'']+'
Get-ChildItem -Path "C:\StrAppers\strAppersFrontend\base44\functions" -Recurse -Filter "entry.ts" -File | ForEach-Object {

## `base44.functions.invoke` (React) vs `entry.ts`

`src/components/boards/ChatSidebar.jsx` calls **`base44.functions.invoke('…')`** for chat, GitHub, mentor, etc. Those names map 1:1 to folders under **`base44/functions`** on the base44 platform. The **HTTP paths** are defined in each folder’s **`entry.ts`** — the inventory file above is the source of truth.
